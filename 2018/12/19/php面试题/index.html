php面试题

后端
  • php

前后端数据交互


  1. form表单：通过form表单以get/post方式提交数据（最原始）
    缺点：在提交的时候可能会导致页面跳转或刷新 用户体验不好
    　　　 单项提交 不知道后台会给出什么响应

  2. ajax：返回数据一般为json形式

php的魔术方法

  • __construct
  • __set()
  • __isset()
  • __unset()
  • __clone()
  • __sleep()

php处理数组的函数

  • array() 　　　　　　　　创建工资
  • in_array() 　　　　　　 判断元素是否存在数组中
  • array_merge() 　　　　 合并数组
  • count() 　　　　　　　　返回数组中元素的个数
  • array_keys() 　　　　　获取数组的key
  • array_values() 　　　　获取数组中的value

php处理字符串的函数

  • trim()　　　　　　移除字符串两侧的空白
  • strlen()　　　　　获取字符串的长度
  • substr()　　　　　截取字符串的一部分
  • strstr()　　　　　找到字符串首字母出现的位置

数据库的三种范式

  • 第一范式：（1NF）无重复的列

  • 第二范式：（2NF）属性完全依赖于主键

  • 第三范式：（3NF）属性不依赖于其他非主属性

    • 消息队列的两种模式

    　　点对点：不可重复消费
    　　发布/订阅：可重复消费

    redis实现消息队列

    　　brpop(消费者)：从队列右侧弹出数据 如果队列中没有数据就堵塞等待
    　　lpush(生产者)：从队列左侧添加数据

    redis应用场景


    1. 会话缓存（最常用）

    2. 消息队列 　比如：支付

    3. 活动排行榜或计数

    4. 发布，订阅消息

    redis 危险命令

    　　flushall 　　清空所有记录,数据库
    　　flushdb 　　 清空数据库
    　　config 　　　客户端连接后可配置服务器
    　　keys　　　　 客户端链接后可查看所有存在的键

    contet-type 常见格式

    　　text/html
    　　text/plain
    　　text/xml
    　　image/gif
    　　image/jpeg
    　　image/png
